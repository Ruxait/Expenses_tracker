# Expenses_tracker
Инструкция по запуску проекта Expenses Tracker

Проект разделён на две папки:
frontend - клиент
backend - сервер

Сначала переходим в папку backend и устанавливаем зависимости
Далее создаём файл .env в папке backend и добавляем переменные окружения:
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/expenses_tracker"
PORT=5000
После настройки запускаем сервер: npm run dev

Переходим в папку frontend и устанавливаем зависимости
Запуск frontend: npm run dev

1. Git
1.1. В общем случае, какие ветки чаще всего используются при работе над проектом?

Ответ:
Обычно есть main (или master) - стабильная версия, develop - для разработки, и feature-ветки (feature/...) под конкретные задачи. Иногда ещё hotfix для срочных исправлений.

1.2. Как отменить коммит?

Ответ:
Если коммит ещё не отправлен, то git reset.
Если уже в удалённом репозитории, то используют git revert, чтобы создать новый коммит, который отменяет изменения.

2. JavaScript
2.1. Что такое Event Loop?

Ответ:
Это механизм, который позволяет JavaScript выполнять асинхронный код. Он следит за стеком вызовов и очередями задач и решает, когда запускать следующий кусок кода.

2.2. Каким образом можно дождаться исполнения нескольких асинхронных операций, не блокируя поток?

Ответ:
С помощью Promise.all (или Promise.allSettled), чтобы запустить операции параллельно и дождаться их завершения без блокировки основного потока.

3. TypeScript
3.1. Чем отличается any от unknown?

Ответ:
any отключает проверку типов полностью.
unknown тоже “любой тип”, но с ним нельзя работать, пока явно не уточнишь тип.

3.2. Что такое function overloading (перегрузка функций)?

Ответ:
Это когда одна функция может иметь несколько вариантов входных параметров, но одна реализация внутри, которая обрабатывает все случаи.

4. Node.js
4.1. Поддерживает ли Node.js Event Loop?

Ответ:
Да, Node.js построен вокруг Event Loop, который позволяет обрабатывать асинхронные операции без блокировки потока.

4.2. При работе со стандартной библиотекой Node.js, как использовать современные функции, работающие на Promise, а не callback'ах?

Ответ:
Через util.promisify или используя версии API с promises, например fs.promises вместо callback-версий.

5. React.js
5.1. Как совершить единовременный HTTP-запрос при рендеринге компонента?

Ответ:
Через useEffect с пустым массивом зависимостей [], чтобы запрос выполнился один раз после первого рендера.

5.2. Что такое key (ключ), и зачем он нужен?

Ответ:
key — это уникальный идентификатор элементов списка. Он помогает React правильно обновлять и перерисовывать список без ошибок и лишних ререндеров.

6. Postgres
6.1. Чем отличаются LEFT JOIN и INNER JOIN?

Ответ:
INNER JOIN возвращает только совпадающие записи.
LEFT JOIN возвращает все записи из левой таблицы и совпадения из правой, а если нет совпадения, то NULL.

6.2. Что такое индекс? Зачем нужно индексировать колонки/поля в таблицах?

Ответ:
Индекс - это структура, которая ускоряет поиск данных в таблице. Он делает SELECT быстрее, но может замедлять INSERT/UPDATE.

7. Отношение к ИИ
7.1. Пользуетесь ли ИИ/LLM, если да, то какими сервисами?

Ответ:
Да, использую ChatGPT и Claude для разработки и поиска решений.

7.2. Каким образом используете ИИ/LLM? Каковы риски?

Ответ:
Использую для генерации кода, объяснения сложных тем, рефакторинга и поиска ошибок.
Риски - возможные ошибки в ответах, избыточная уверенность модели и потеря критического мышления.

7.3. Как бороться с перечисленными вами рисками?

Ответ:
Проверять результат вручную, не доверять слепо, тестировать код, сверяться с документацией и использовать ИИ как помощника, а не как источник истины.
